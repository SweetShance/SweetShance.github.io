Redis为什么这么快？请从至少5个方面详细解释。

  • reid 是直接从内存读取的减少了i/o读取时间
  • redise 是key-value存储，且包含 多种数据结构，数据结构简单，性能高
  • reid 是单线程的 避免多线程的上下文切换
  • redis是非阻塞I/O： 一步处理网络请求，不阻塞主线程
  • redis 是C写的，更接近底层

RDB和AOF两种持久化方式各自的优缺点是什么？在生产环境中如何选择和配置？

  • RDB：是采用快照的形式每隔一段时间将数据写入到 磁盘中

  • AOF：是将reids的操作追加 到一个日志文件中，然后通过一个线程从日志文件中恢复数据
    RDB 优点： 恢复快，备份小 缺点： 数据一致性差，可能会丢失，fork子进程的时候 容易阻塞
    AOF 优点： 数据一致性高 缺点： 占用磁盘空间大， 数据恢复慢

主从复制、哨兵模式、Cluster集群三种高可用方案的区别和适用场景是什么？

  • 主从复制：主库写入从节点复制，实现读写分离，适用于访问频率高的情况，读多写少的情况。
  • 哨兵机制： 从节点会从主节点复制，主节点挂了，会从从节点选一个座位主节点，适用于稳定的系统，故障自动转移。
  • Cluster集群： 集群是多个节点组成的，每个节点都保存了数据，数据是分布的，适用于大数据量高并发的情况。
